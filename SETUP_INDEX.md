## 🆘 TROUBLESHOOTING BY PROBLEM

### Installation Issues
**Problem**: "composer: command not found"  
→ Solution: Use `php composer.phar install`  
→ Guide: [INSTALLATION.md - Troubleshooting](INSTALLATION.md#troubleshooting)

**Problem**: "PHP not recognized"  
→ Solution: Refresh PATH in PowerShell  
→ Guide: [REFERENCE.md - Quick Reference](REFERENCE.md)

**Problem**: "Database connection error"  
→ Solution: Check .env configuration  
→ Guide: [INSTALLATION.md - Database Setup](INSTALLATION.md#step-5-run-database-migrations)

### Development Issues
**Problem**: "Port 8000 already in use"  
→ Solution: Use different port  
→ Command: `php artisan serve --port=8001`

**Problem**: "SQLSTATE database error"  
→ Solution: Create database file  
→ Guide: [REFERENCE.md - If Something Goes Wrong](REFERENCE.md#-if-something-goes-wrong)

**Problem**: "npm not found"  
→ Solution: Optional, not required  
→ Info: [REFERENCE.md - Optional Node.js](REFERENCE.md#-optional-install-nodejs-for-cssjs)

## ✅ VERIFICATION CHECKLIST

Before you start, verify you have:
- [ ] PHP 8.3+ (run `php -v`)
- [ ] Composer PHAR downloaded (file `composer.phar` exists)
- [ ] Project files (app/, database/, routes/, etc.)
- [ ] Configuration template (`.env.example`)

After installation, verify:
- [ ] `vendor/` directory exists
- [ ] `composer.lock` file created
- [ ] `.env` configured with credentials
- [ ] `php artisan key:generate` runs
- [ ] `php artisan migrate` completes
- [ ] `php artisan serve` starts
- [ ] http://localhost:8000 loads
